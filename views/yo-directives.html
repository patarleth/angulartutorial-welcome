directives, building a more symantic web

Angularjs and it's templating framework is built (almost) entirely on a single concept, the

    directive.

a simple and often used example of a directive is ng-repeat

    <div ng-repeat="headline in espnnow.feed">

a directive enables HTML extentions and a templates behaviour by coupling together data, code (controllers) and view.

in the ng-repeat directive the data would be the string "headline in espnnew.feed"
the code for the directive is responsible for figuring out the data binding, aka what espnnow.feed is looping over the array.
the contents of the div are used as the view to create the output

directives are not limited to attributes on existing tags you can create entirely new HTML tags and do tricks like -

  <dl class="dl-horizontal" ng-repeat="next in espnnow.feed">
    <dt><span ng-show="next.type != 'BlogEntry'">{{next.type}}</span><span ng-show="next.type == 'BlogEntry'">{{next.blogName + ' blog'}}</span></dt>
    <dd>
      <espn-headline headline="next"></espn-headline>
    </dd>
  </dl>
