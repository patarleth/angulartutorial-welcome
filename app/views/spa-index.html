index.html

Application specific markup does NOT belong in index.html.

The index.html should be responsible for loading your css/javascript files and "bootstraping" angularjs framework.

  • the markup and scripts in index.html are responsible for browser mitigation and global app settings like google analytics.
  • there are two important tags in main index.html page -
    • <body ng-app="ngTutorialApp"> - angularjs "bootstrap" designates the root element managed by angularjs
    • <div class="container" ng-view=""></div> - where the views for you application will be placed by angulars routing mechanism
  • script tags are surrounded by html comments indicating where the script originated from
      <!-- bower:js --> - indicates the following script tags are dependencies managed by bower.
      <!-- build:js({.tmp,app}) scripts/scripts.js --> - following script tags belong to the application
  • the yeoman generated html comments are used by grunt to 'build' minified distribution
